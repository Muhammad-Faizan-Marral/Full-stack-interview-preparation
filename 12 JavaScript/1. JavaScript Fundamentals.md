# 1. What is JavaScript?

JavaScript (JS) is a **high-level, interpreted, dynamically typed programming language** used to create interactive web pages and applications.

Originally, JavaScript was designed to run only in web browsers, but today it can also run on servers using **Node.js**.

### Features

- Lightweight
- Object-Oriented
- Event-Driven
- Cross-platform
- Dynamically Typed
- Supports Functional Programming
- Asynchronous Programming

### Example

```javascript
console.log("Hello, World!");
```

---

# 2. How does JavaScript work?

JavaScript executes inside a **JavaScript Engine**.

Popular engines include:

- V8 (Chrome & Node.js)
- SpiderMonkey (Firefox)
- JavaScriptCore (Safari)

### Execution Process

1. Browser loads HTML.
2. CSS is parsed.
3. JavaScript engine reads JS code.
4. Code is parsed.
5. Code is compiled (JIT Compilation).
6. Machine code executes.

### JavaScript Runtime Includes

- Call Stack
- Heap Memory
- Web APIs
- Callback Queue
- Event Loop

Example:

```javascript
console.log("Start");

setTimeout(() => {
    console.log("Timer");
}, 0);

console.log("End");
```

Output:

```
Start
End
Timer
```

# 3. What is ECMAScript?

ECMAScript (ES) is the **official specification** that defines how JavaScript should work.

JavaScript is an implementation of ECMAScript.

### Popular Versions

| Version | Features |
|----------|----------|
| ES5 | Strict Mode |
| ES6 (2015) | let, const, arrow functions, classes |
| ES7 | includes() |
| ES8 | async/await |
| ES9 | Rest/Spread |
| ES10 | flat() |
| ES11 | Optional Chaining |
| ES12 | replaceAll() |
| ES13+ | New improvements |

# 4. JavaScript vs TypeScript

| JavaScript | TypeScript |
|------------|------------|
| Dynamically typed | Statically typed |
| No compilation | Compiled to JavaScript |
| No interfaces | Supports interfaces |
| Easier to start | Better for large projects |
| Runtime errors | Catches many errors during development |

JavaScript:

```javascript
let age = 20;
age = "Twenty";
```

TypeScript:

```typescript
let age: number = 20;

// Error
age = "Twenty";
```

# 5. Primitive vs Reference Data Types

## Primitive Types

Stored by value.

- String
- Number
- Boolean
- Undefined
- Null
- Symbol
- BigInt

Example:

```javascript
let a = 10;
let b = a;

b = 20;

console.log(a); // 10
console.log(b); // 20
```

---

## Reference Types

Stored by reference.

- Object
- Array
- Function
- Date
- Map
- Set

Example:

```javascript
let person1 = {
    name: "John"
};

let person2 = person1;

person2.name = "Mike";

console.log(person1.name);
// Mike
```

---

# 6. What is Dynamic Typing?

JavaScript variables do not have fixed types.

A variable can store different types of values.

```javascript
let value = 10;

value = "Hello";

value = true;
```

---

# 7. What is Type Coercion?

JavaScript automatically converts one data type into another.

Example:

```javascript
console.log("5" + 2);
```

Output

```
52
```

Example:

```javascript
console.log("5" - 2);
```

Output

```
3
```

# 8. == vs ===

## == (Loose Equality)

Converts types before comparison.

```javascript
5 == "5"
```

Output

```
true
```

---

## === (Strict Equality)

No type conversion.

```javascript
5 === "5"
```

Output

```
false
```

Always prefer `===`.

# 9. null vs undefined

## undefined

Means a variable has been declared but not assigned.

```javascript
let x;

console.log(x);
```

Output

```
undefined
```

---

## null

Represents intentional absence of value.

```javascript
let user = null;
```

---

Comparison

| null | undefined |
|------|-----------|
| Intentional | Default |
| Object type (historical bug) | Undefined type |

# 10. NaN

NaN means **Not a Number**.

```javascript
console.log(Number("abc"));
```

# 11. isNaN() vs Number.isNaN()

## isNaN()

Performs type conversion.

```javascript
isNaN("abc");
```

Output

```
true
```

---

## Number.isNaN()

No type conversion.

```javascript
Number.isNaN("abc");
```

Output

```
false
```

```javascript
Number.isNaN(NaN);
```

Output

```
true
```

# 12. Truthy and Falsy Values

## Falsy Values

Only these values are falsy:

```javascript
false
0
-0
0n
""
null
undefined
NaN
```

Everything else is **truthy**.

Example:

```javascript
if ("Hello") {
    console.log("Truthy");
}
```

# 13. typeof Operator

Returns the type of a value.

```javascript
typeof 10
```

Output

```
"number"
```

Examples:

```javascript
typeof "Hello"
// string

typeof true
// boolean

typeof undefined
// undefined

typeof null
// object (historical bug)

typeof {}
// object

typeof []
// object

typeof function(){}
// function
```

# 14. instanceof Operator

Checks whether an object is created from a constructor.

```javascript
const arr = [];

console.log(arr instanceof Array);
```

Output

```
true
```

Example:

```javascript
new Date() instanceof Date
```

Output

```
true
```

---

# 15. Object.is()

Compares two values more precisely than `===`.

Examples:

```javascript
Object.is(NaN, NaN)
```

Output

```
true
```

```javascript
NaN === NaN
```

Output

```
false
```

Another example:

```javascript
Object.is(+0, -0)
```

Output

```
false
```

# 16. Symbol Data Type

A Symbol is a unique and immutable primitive value.

```javascript
const id = Symbol();
```

Even if descriptions are the same:

```javascript
const s1 = Symbol("id");
const s2 = Symbol("id");

console.log(s1 === s2);
```

Output

```
false
```

Useful for unique object property keys.

# 17. BigInt

BigInt stores integers larger than the safe integer limit.

Example:

```javascript
const big = 123456789012345678901234567890n;
```

Without BigInt:

```javascript
Number.MAX_SAFE_INTEGER
```

```

```

# 18. parseInt() vs Number()

## parseInt()

Parses integers.

```javascript
parseInt("100px")
```

Output

```
100
```

---

## Number()

Converts the whole value.

```javascript
Number("100px")
```

Output

```
NaN
```

Comparison

| parseInt() | Number() |
|------------|-----------|
| Stops at invalid character | Converts entire string |
| Returns integer | Returns integer or decimal |
| Ignores trailing text | Returns NaN if invalid |

---

# 19. parseFloat()

```
10.55
```

```javascript
parseFloat("100")
```

Output

```
100
```

# 20. Number Methods

## Number.isInteger()

```javascript
Number.isInteger(10);
```

Output

```
true
```

---

## Number.isFinite()

```javascript
Number.isFinite(100);
```


---

## Number.isSafeInteger()

```javascript
Number.isSafeInteger(100);
```

Output


---

## toFixed()

Rounds to fixed decimal places.

```javascript
let num = 10.567;

console.log(num.toFixed(2));
```

Output

```
10.57
```

---

## toPrecision()

Formats to a specified precision.

```javascript
let num = 123.456;

console.log(num.toPrecision(4));
```

Output

```
123.5
```

---

## toString()

Converts a number to a string.

```javascript
let num = 100;

console.log(num.toString());
```

Output

```
"100"
```

---

## Number()

Converts values to numbers.

```javascript
Number("123");
```

Output

```
123
```
